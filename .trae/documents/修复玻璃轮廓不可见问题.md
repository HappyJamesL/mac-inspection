## 修复玻璃轮廓不可见问题

### 问题分析
玻璃轮廓不可见的主要原因是API返回空数据导致`glassConfig.size`被设置为空对象，从而无法计算正确的轮廓尺寸。另外，可能需要提高轮廓的可见性。

### 解决方案
修改`CanvasStage.vue`文件，确保玻璃尺寸有默认值，并优化轮廓的可见性。

### 具体修改点

1. **添加玻璃尺寸默认值**：
   - 在`loadPanelData`函数中，当API返回空数据时，使用默认尺寸
   - 默认尺寸保持为920000um x 730000um
   - 确保`glassConfig.size`始终有有效值

2. **优化轮廓可见性**：
   - 增加轮廓线宽至3px
   - 提高透明度至1.0
   - 考虑添加轻微的阴影效果
   - 可以使用更醒目的颜色（如黄色）

3. **调整绘制顺序**：
   - 确保轮廓绘制在正确的层级
   - 验证`mainLayer.destroyChildren()`不会影响后续绘制

4. **调试信息**：
   - 添加控制台日志，输出玻璃尺寸和轮廓路径
   - 方便调试和验证

### 修改策略
- 先解决尺寸默认值问题
- 再优化轮廓可见性
- 最后添加调试信息

### 预期效果
- 玻璃轮廓在所有情况下都可见
- 尺寸始终为920mm x 730mm
- 轮廓清晰醒目
- 便于调试和维护

### 关键代码优化
```javascript
// 在loadPanelData函数中添加默认值
const data = await getProductLayout(props.selectedProductId);
glassConfig.size = data.glass_size || { w: 920000, h: 730000 };
glassConfig.panels = data.panels || [];
glassConfig.maskRule = data.mask_rule || { cover_x: 5, cover_y: 6, full_shot: 1 };

// 优化轮廓样式
mainLayer.add(new Konva.Line({
  points: screenPath,
  stroke: '#ffff00', // 黄色，更醒目
  strokeWidth: 3, // 更粗的线宽
  closed: true,
  listening: false,
  opacity: 1.0, // 完全不透明
  shadowColor: '#ffff00',
  shadowBlur: 2,
  shadowOffsetX: 1,
  shadowOffsetY: 1,
  shadowOpacity: 0.5
}));
```
